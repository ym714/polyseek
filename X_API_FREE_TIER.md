# X（Twitter）API 無料プランについて

## ⚠️ 現状（2024-2025年）

X（旧Twitter）APIの無料プランは**非常に制限的**です：

### 無料プランの制限

1. **ツイート検索（読み込み）**
   - 15分あたりのリクエスト数が非常に少ない
   - 月間のツイート取得数に厳しい制限
   - 商用利用不可

2. **API制限**
   - 基本的な機能のみ
   - 高度な検索機能は制限される可能性

3. **料金プラン**
   - **Free**: 非常に制限的（ほぼ使えない）
   - **Basic ($100/月)**: 基本的な機能
   - **Pro ($5,000/月)**: フル機能

## 💡 代替案

### 1. Reddit API（完全無料・推奨）

Reddit APIは**完全無料**で、商用利用も可能です：

```bash
# Reddit APIは無料で使えます
export REDDIT_CLIENT_ID="your-client-id"
export REDDIT_CLIENT_SECRET="your-client-secret"
export REDDIT_USER_AGENT="polyseek/1.0"
```

**メリット**:
- ✅ 完全無料
- ✅ 商用利用可能
- ✅ レート制限が緩い
- ✅ コミュニティの議論を取得可能

### 2. ニュースAPI（既に実装済み）

News APIは無料プランがあります：
- 開発用: 1日100リクエストまで無料
- 商用: 有料プランあり

### 3. RSSフィード（完全無料）

主要ニュースサイトのRSSフィードを使用：
- Google News RSS
- 各ニュースサイトのRSS
- 完全無料、制限なし

## 🎯 推奨アプローチ

### 現在の実装状況

1. ✅ **News API**: 実装済み（無料プラン利用可能）
2. ⚠️ **X API**: 実装済み（無料プランは制限的）
3. 🔄 **Reddit API**: 実装推奨（完全無料）

### 最適な組み合わせ

**無料で使う場合**:
- News API（無料プラン）
- Reddit API（完全無料）
- RSSフィード（完全無料）

**X APIを使う場合**:
- Basicプラン（$100/月）以上が必要
- または、無料プランで試用（制限あり）

## 🚀 次のステップ

### Reddit統合を実装する（推奨）

Reddit APIは完全無料で、X APIよりも実用的です：

1. **Reddit APIキー取得**（無料）
   - https://www.reddit.com/prefs/apps にアクセス
   - アプリを作成（約5分）
   - Client IDとSecretを取得

2. **実装**
   - `signals_client.py`に`RedditSignalProvider`を追加
   - 設定を追加

3. **メリット**
   - 完全無料
   - 商用利用可能
   - コミュニティの深い議論を取得可能
